SETUP INSTRUCTIONS
==================

Prerequisites:
- Docker & Docker Compose installed
- Google Gemini API key: ALREADY CONFIGURED IN .env

Installation:
1. Run: ./start.sh
2. Access at http://localhost:3000

DEPLOYING ON NAS WITH DOCKGE:
See DOCKGE_QUICKSTART.txt for step-by-step instructions

Your Gemini API key is already configured in .env file.

Default Ports:
- Frontend: 3000
- Backend: 8000
- Caddy: 80/443

Themes:
- 6 themes available (3 static, 3 animated)
- See THEMES.txt for details
- Change theme in portfolio editor

Commands:
- Start: docker-compose up -d
- Stop: docker-compose down
- Logs: docker-compose logs -f
- Restart: docker-compose restart

Environment (.env):
- GEMINI_API_KEY: Your Google Gemini API key (required)
- JWT_SECRET: Auto-generated by start.sh
- POSTGRES_PASSWORD: Auto-generated by start.sh

Backups:
- Database: docker-compose exec postgres pg_dump -U aptifolio aptifolio > backup.sql
- Uploads: tar -czf uploads.tar.gz uploads/

Production Setup:
1. Edit caddy/Caddyfile with your domain
2. Point DNS A record to your server
3. Restart Caddy: docker-compose restart caddy
4. SSL certificates automatic via Let's Encrypt

Troubleshooting:
- Check logs: docker-compose logs backend
- Verify API key in .env
- Ensure ports 80, 443, 3000, 8000 are available
- Database reset: docker-compose down -v && docker-compose up -d
